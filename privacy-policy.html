Privacy Policy for MavDroid

MavDroid is provided by SK-Delta Works. This page explains how the app handles data and device access when used as a MAVLink bridge and RTSP streaming tool for USB-connected flight controllers, including ArduPilot and PX4.

App: MavDroid
Developer: SK-Delta Works
Last Updated: 2026-04-15

Overview

MavDroid is an Android app that provides MAVLink communication bridging and RTSP video streaming for flight controllers connected to an Android device via USB. It is intended for experiments, testing, and development use.

Main Purpose

Bridge MAVLink communication between a ground station device and a USB-connected flight controller.

Video Streaming

Stream camera video from the Android device through RTSP for local or VPN-based receiving environments.

Supported Targets

USB-connected flight controllers using ArduPilot or PX4 based systems.

Use Environment

A compatible USB connection, proper communication settings, and a network path between sender and receiver are required.

Data Handling

The app is designed to focus on device-side communication and streaming functions. The following points describe how data is handled.

  • No account registration required
    MavDroid does not require the user to create an account to use its core features.
  • No sale of user data
    MavDroid does not sell user data to third parties.
  • No third-party advertising
    MavDroid does not display third-party ads in the app.
  • Local function-oriented use of permissions
    USB connection, network access, and camera access are used only for the core functionality of MAVLink bridging and RTSP streaming.
  • Local processing by default
    Unless otherwise stated, data processed by the app is handled locally on the device for its core functionality.

Permissions and Access

MavDroid may require access to device features depending on how the app is used.

USB ACCESS

Used to communicate with supported flight controllers connected to the Android device.

NETWORK ACCESS

Used for MAVLink network bridging and RTSP streaming between sender and receiving devices.

CAMERA ACCESS

Used only when the RTSP video streaming function is enabled by the user.

  • Communication requirements
    The Android device connected to the flight controller and the receiving device must be able to communicate with each other over the network.
  • Supported network layouts
    Use the app within the same Wi-Fi network, or when operating outdoors or across external networks, connect both sides through a VPN.
